02 CONTEXT · WHY THIS GUIDE
why cloudflare pages

이 가이드의 선택 — Cloudflare Pages.

Claude Code, Codex 등으로 사이트를 만들고 나면 부딪히는 다음 단계 — 이 코드 어떻게 사람들한테 보여주지? 이 가이드는 그 답으로 Cloudflare Pages를 다룬다. 무료에 가깝고, GitHub과 한 번 연결로 자동화되고, 전 세계 어디서나 빠르다.

01 · free
무료 · 카드 정보 없이 시작
개인·내부 프로젝트는 무료 플랜으로 충분. 빌드 500회/월, 동시 빌드 1개 — 일반 사용에 넉넉.
02 · auto
한 번 연결로 자동화
설정 한 번이면, 이후 code 변경 → push → 배포가 자동. CI/CD 별도 구성 없이 GitOps 흐름.
03 · fast
전 세계 CDN
한국에서 빌드해도 전 세계 어디서나 가까운 노드에서 즉시 로딩. 로컬 서버 신경 안 써도 됨.
검토 권장
다만 '최적의 답'은 그때그때 바뀔 수 있다. "내 프로젝트엔 Cloudflare Pages가 적합해 보이는데, 더 나은 선택지가 있을까?" — AI에게 한 번 물어보고 시작하는 게 안전. (참고: Cloudflare는 오랫동안 이 분야의 강자였기 때문에 결론은 비슷할 가능성이 높다.) 실제로 사내 '삼삼한 임대인'은 처음부터 가비아에서 유료 도메인을 사서 시작했다 — 상황이 다르면 답도 다르다. 그런 갈림길은 그때 가서 AI에게 하나씩 물어가며 해결하면 된다.

04 BEFORE THAT · DESIGN
before that

그전에 — 디자인은 어떻게?

디자인을 할 줄 몰라도 시작에는 문제가 없다. 초보 단계에서는 아래 두 가지면 충분하다.

01 디자인 가이드 받아 먹이기 — https://getdesign.md 같은 Design.md 공유 사이트에서 마음에 드는 테마의 md 파일을 받아, "이 디자인 가이드대로 만들어줘"라고 첨부한다.
02 참고 자료 던지기 — md 방식이 아니어도 된다. 원하는 사이트의 PDF나 스크린샷을 주고 "이 느낌으로"라고만 해도 어느 정도 알아서 잘해준다.
사내 실례
삼삼뉴스가 실제로 이 방식이다 — 삼삼엠투 홈페이지 PDF를 따서 "참고"만 시킨 결과물.
scope
좋은 디자인을 만드는 법, 개발자와 협업하기 좋게 디자인에 AI를 활용하는 법은 그것만으로 별도 강의가 필요한 주제다 — 오늘은 "그럴듯하게 시작하는 법"까지만 다룬다.

05 DECISIONS · THE THREE
decisions at a glance

결정 세 가지 — 한눈에.

결정 ①
무엇을 만들지 — 딥인터뷰로
미리 공부할 것 없다. 인터뷰 → 계획 합의 → 제작 흐름에서 프레임워크 같은 기술 선택까지 AI와 함께 결정된다. 첫 연습은 파일럿 수준부터 가볍게.
결정 ②
어떻게 배포 — GitHub 연동
push = 자동 빌드 + 배포. 코딩 AI의 표준 동작(commit/push)과 그대로 맞물려 이 가이드의 선택. (일회성 드래그 업로드, CLI, API 방식도 있다 — 필요해지면 그때 AI에게.)
결정 ③
빌드 설정 — AI에게 미리 확인
Cloudflare 화면에 입력할 빌드 명령어·출력 폴더를 코딩 AI에게 표로 받아 둔다. 입력 단계에서 그대로 베껴 넣으면 끝.

바로 만들지 말고, 먼저 deep interview로 아이디어를 구체화해줘.
인터뷰 중 범위가 내 시간 대비 커지면, 줄여서 가자고 먼저 제안해줘.
기술 선택(프레임워크 등)은 지금 시점 기준으로 추천하되, 학습 시간이
'없음'이면 새로 공부할 필요 없는 쪽으로. 특정 버전 요구사항이 있으면

무엇을 어떻게 만들지 계획을 보여주고, 내 OK를 받은 뒤에 시작.

06 AFTER BUILD · SETTINGS CHECK
after build

제작이 끝나면 — 빌드 설정 확인.

보통은 코딩 AI가 작업을 마무리하면서 이 값들을 알아서 알려준다. 혹시 안 알려줬을 때를 위해 적어 둔다 — Cloudflare 화면에 들어가기 전에 아래 프롬프트로 표를 받아 두면 된다.

PROMPT 02 · 빌드 설정 자동 확인 — 제작이 끝난 프로젝트에서 그대로 실행
현재 작업 중인 프로젝트의 package.json을 확인하고
다음 정보를 표로 정리해줘:

| 항목 | 값 |
|---|---|
| Framework | (dependencies로 추정) |
| Build command | scripts.build 값 |
| Output directory | 빌드 결과물 폴더명 |
| Node version | engines 또는 .nvmrc |
| Package manager | npm / yarn / pnpm |

만약 모노레포(루트가 아닌 하위 폴더에 프로젝트)이면
그 경로도 함께 알려줘.

이 정보를 그대로 Cloudflare Pages 빌드 설정에 입력할 예정이야.
tip
출력된 표를 스크린샷이나 메모로 가지고 있으면, Cloudflare 설정 입력 단계에서 그대로 베껴 넣을 수 있다.

14 AUTOMATION · GITOPS
the real magic

이제부터 push = 배포.

한 번 연결되면 끝. main 브랜치에 어떤 식으로든 push가 일어나면 Cloudflare가 자동으로 다시 빌드해서 배포한다.

trigger 01
로컬 / Codespaces
Claude Code, Codex 등 코딩 AI에서 git commit + push. 가장 흔한 경우.
trigger 02
GitHub 웹
브라우저에서 파일 직접 편집·저장. commit이 자동 생성되며 같은 효과.
trigger 03
GitHub Actions
자동 워크플로우가 commit (스케줄·자동 데이터 갱신 등). 똑같이 트리거됨.
flow
세 경우 모두 똑같은 방식으로 빌드된다. 평균 1분 내 반영. 빌드 실패 시 Cloudflare 대시보드에 로그 + 이메일 알림.

15OPERATIONS · BUILD BUDGET
build budget

무료 빌드 500회 — 아끼는 스위치 두 개.

push = 배포는 편하지만, 모든 push가 빌드 1회를 쓴다. 무료 플랜은 월 500회(하루 약 16회) — 보통은 넉넉하지만, 사이트와 무관한 변경까지 빌드를 돌릴 이유는 없다.

스위치 01 · 그때그때
커밋 메시지로 스킵
커밋 제목 앞에 [CI Skip](또는 [Skip CI])을 붙이면 그 push는 배포를 건너뛴다. 메모·문서만 고쳤을 때 — AI에게 "이번 커밋은 배포 스킵해줘"라고 말하면 알아서 붙인다. 잔손질(chore)성 커밋이면 말하지 않아도 AI가 알아서 붙여 주는 경우가 많다 — 안 붙어 있을 때만 위 한마디면 충분.
스위치 02 · 자동
Build watch paths
프로젝트 Settings → Build → Build watch paths. 지정 경로가 바뀔 때만 빌드(기본은 모든 변경). docs/* 같은 폴더를 exclude에 넣어두면 그쪽 변경은 자동으로 스킵된다.
EXAMPLE PROMPT 세팅 마지막에 한 번 — 코딩 AI에게 맡기기
내 repo 구조를 보고, 사이트 빌드와 무관한 경로(문서, 메모 등)가 있는지 확인해줘.
있다면 Cloudflare Pages의 Build watch paths에 넣을 include / exclude 값을 추천해줘.
설정 위치: 프로젝트 Settings → Build → Build watch paths.
heads-up
한 번의 push에 변경 파일 3000개 이상 또는 커밋 20개 이상이면 경로 설정과 무관하게 빌드된다 — 예외 케이스로만 기억해 두면 충분.

16 OPERATIONS · DEBUGGING
pitfalls & debugging

막혔을 때 — AI한테 그대로 던지기.

처음 배포할 때 자주 부딪히는 함정 세 가지. 알고 있으면 쉽게 피하고, 막혀도 빠르게 해결.

trap 01
Node 버전 mismatch
도구에 따라 그때그때 최신 Node 버전을 요구하기도 한다. 버전 지정이 필요하면 환경변수 NODE_VERSION=24 식으로 지정 (숫자는 요구 버전에 맞춰).
trap 02
Root directory 누락
사이트 코드가 하위 폴더면 Root directory 명시 필수. 누락 시 package.json not found.
trap 03
환경변수 노출
API 키 등 secret을 .env에 두고 commit하면 안 됨. Cloudflare 대시보드 Settings에서 별도 설정.
EXAMPLE PROMPT · 빌드 실패 디버깅 — 로그 통째로 붙여넣고 분석 받기 
Cloudflare Pages 빌드가 실패했어. 아래는 빌드 로그의 마지막 100줄이야:

---
[빌드 로그 붙여넣기]
---

다음을 분석해줘:
1. 정확한 실패 원인 (어느 단계, 어떤 에러)
2. 가능한 해결 방법 (우선순위 순으로)
3. 코드 수정이 필요하면 수정할 파일과 변경 내용
4. Cloudflare Pages 설정 변경이 필요하면 어느 항목을 어떻게 바꿔야 하는지

내 프로젝트: [프레임워크 + 간단 설명]
GitHub repo: [repo URL 또는 구조 설명]
