PART 1

내 기준대로 일하는 AI를 이루는 기본 요소

AI가 나답게 일하려면 기준, 기억, 반복 절차, 역할, 도구 연결이 구분되어야 합니다.

Instruction(지침)

AI가 답변할 때 항상 따라야 할 역할, 행동 기준, 금지선, 참고 정보를 적습니다

지침에 들어갈 4가지

역할 부여, 해야 할 것, 하지 말아야 할 것, 항상 참고할 정보

언제 쓰나

AI가 매번 같은 역할과 기준으로 답변해야 할 때.

AI가 보는 방식

답변 전에 먼저 읽고, 해야 할 일과 금지선, 참고 자료를 확인합니다.

# AGENTS.md 또는 CLAUDE.md 예시 # 총 주방장 운영 지침 ## 역할 - 전체 기준을 정하고 담당 Agent에게 일을 나눈다. - 손님 요청, 금지 재료, 완성 시간을 끝까지 확인한다. ## 해야 할 것 - 주문 조건을 먼저 확인한다. - 재료 손질, 양식, 찜 요리 담당을 나눈다. - 간, 익힘, 플레이팅을 마지막에 점검한다. - 정보가 부족하면 다시 질문한다. ## 하지 말아야 할 것 - 손님이 말하지 않은 재료를 임의로 추가하지 않는다. - 알레르기나 금지 재료를 가볍게 넘기지 않는다. ## 항상 참고할 정보 - 손님 취향, 알레르기, 금지 재료를 먼저 확인한다. - 이전 주문 기록과 오늘 가능한 재료를 기준으로 판단한다.

Memory(기억)

총 주방장이 계속 참고하면 좋은 손님 취향과 반복 정보를 남깁니다

주방의 반복 기억

손님 취향, 자주 쓰는 재료, 선호하는 마무리 방식

언제 쓰나

단골 손님 취향처럼 매번 다시 말하지 않아도 좋은 정보를 저장할 때.

지침과 차이

Instruction은 반드시 지킬 운영 기준, Memory는 참고하면 좋은 반복 기억입니다.

# memory.md 예시 # 총 주방장이 참고할 반복 기억 - 단골 손님은 마늘 향은 좋아하지만 너무 맵게 먹지 않는다. - 설명할 때는 어려운 조리 용어보다 쉬운 말로 먼저 안내한다. - 결과를 낼 때는 “요리 이름, 맛의 특징, 확인한 기준” 순서로 정리한다. - 예시는 토마토 파스타처럼 익숙한 요리로 들면 이해가 빠르다. - 단, 알레르기와 금지 재료는 Memory에만 맡기지 말고 매번 요청에서 다시 확인한다.

Context(대화의 맥락)

AI와 주고받은 질문과 답, 수정 요청, 첨부 자료가 이번 답변의 맥락이 됩니다
나비개발자에게 AI 기초를 쉽게 설명하는 자료를 만들고 싶어.
AI좋아요. 먼저 좋은 프롬프트, 기억, 지침, 역할을 나눠 설명해볼게요.
나요리사 예시를 쓰되, “비유”라는 단어는 화면에 넣지 말아줘.
AI그러면 요리 흐름은 자연스럽게 녹이고, 화면 문구는 교육 주제 중심으로 정리하겠습니다.

Context란?

지금 대화에서 오간 말의 흐름입니다. AI는 방금 전 질문과 답변, 수정 요청을 함께 보고 다음 답을 만듭니다.

왜 중요할까?

“비개발자 대상”, “쉬운 말”, “화면에는 비유라는 단어를 넣지 않기” 같은 조건이 맥락 안에 있어야 결과가 맞춰집니다.

실무 팁

대화가 길어지면 중요한 조건을 새 요청에 다시 적습니다. 예: “앞에서 말한 조건 유지: 쉬운 단어, 자연스러운 요리 예시.”

Context Window(참고 범위)

대화와 자료가 길어지면 모든 데이터가 그대로 남지 않고 일부가 유실될 수 있습니다
참고 범위

Context Window는 AI가 한 번에 참고할 수 있는 정보의 범위입니다. 이 범위를 넘으면 대화는 압축되고, 세부 데이터 일부는 유실될 수 있습니다.

1. 대화가 쌓임요청과 답변이 계속 이어져요

대화를 이어갈수록 이전 요청, 답변, 수정 사항, 참고 자료가 Context Window 안에 쌓입니다.

Context Window현재 참고 막대
초기 대화
추가 질문
참고 자료
여유

대화가 길어질수록 참고 막대가 점점 채워집니다.

2. 기억을 압축함계속 대화하기 위해 요약돼요

범위를 넘으면 모든 내용을 그대로 들고 가지 못해서, 이전 대화가 압축된 요약본으로 바뀝니다.

Context Window압축 후
압축된 이전 대화
남은 핵심
유실 가능 정보
여유

세부 표현, 예시, 작은 조건은 압축 과정에서 빠질 수 있습니다.

3. 압축된 상태로 이어감요약된 기억을 바탕으로 계속 답해요

압축 이후에도 대화는 계속 이어지지만, 빠진 세부 조건은 AI가 놓칠 수 있습니다.

Context Window대화 지속
압축본
새 요청
이어지는 답변
여유

대화는 이어지지만, 압축에서 빠진 정보는 약해질 수 있습니다.

실무 팁
계속 유지되어야 하는 기준은 대화에만 남기지 말고 Memory에 저장합니다.
예: “앞으로 고객 안내문은 정중하고 짧게, 핵심 일정과 다음 행동을 꼭 포함해줘.”

Skill(업무 절차)

SKILL.md에 반복되는 업무 순서와 검토 기준을 저장합니다

반복 업무 절차

언제 쓰고, 어떤 순서로 하고, 어떤 형식으로 끝낼지

언제 쓰나

토마토 파스타처럼 재료 준비, 도구 사용, 조리 순서가 정해진 일을 반복할 때.

좋은 점

짧게 요청해도 매번 같은 순서와 기준으로 요리가 진행됩니다.

# SKILL.md 예시 --- name: tomato-pasta description: 내가 선호하는 순서로 토마토 파스타 만들기 --- 1. 손님 수, 금지 재료, 완성 시간을 확인한다. 2. 냄비, 팬, 집게, 체, 접시를 준비한다. 3. 물을 끓이고 면을 삶는다. 4. 팬에 마늘, 토마토소스, 면수를 넣어 소스를 만든다. 5. 면을 소스에 버무리고 간과 플레이팅을 확인한다.

Agent(역할을 맡은 AI)

AGENT.md에 역할, 위임 기준, 결과 확인 방식을 적습니다

역할을 맡은 AI

전체를 지휘하거나, 특정 담당 일을 맡아 수행함

언제 쓰나

총괄 역할이 여러 담당 Agent에게 일을 나누고 결과를 모아야 할 때.

Skill과 차이

Skill은 반복 절차, Agent는 책임과 판단 기준을 가진 역할입니다.

# AGENT.md 예시 # 총 주방장 Agent ## 역할 - 손님 요청과 금지 재료를 확인한다. - 담당 Agent에게 일을 나누고 결과를 모은다. - 최종 코스의 맛, 순서, 완성 시간을 책임진다. ## 위임 기준 - 재료 준비 → 재료 손질 담당 Agent - 파스타/스테이크 → 양식 담당 Agent - 찜 요리 → 찜 요리 담당 Agent ## 진행 순서 1. 손님 취향, 알레르기, 완성 시간을 확인한다. 2. 담당 Agent에게 역할과 결과물 형식을 전달한다. 3. 중간 결과를 점검하고 수정 요청을 보낸다. 4. 최종 메뉴 구성과 플레이팅을 확인한다. ## 하지 말아야 할 것 - 담당 Agent 결과를 확인 없이 내지 않는다. - 금지 재료나 알레르기 정보를 무시하지 않는다.

PART 2

불편했던 사례로 다시 보기

AI가 이상하게 군다고 느꼈던 장면은 대부분 “요청, 기준, 기억, 확인 방식” 문제로 설명할 수 있습니다.

“하지 말라고 했는데 자꾸 말을 안 들어요.”

한 번의 요청과 계속 지켜야 하는 지침을 구분해야 합니다
나고객에게 보낼 배송 지연 안내문이야. 변명처럼 들리는 말은 쓰지 마.
AI물류 상황상 불가피하게 지연되었으니 양해 부탁드립니다...
나변명처럼 쓰지 말라고 했잖아.
AI죄송합니다. 금지 표현을 빼고 다시 작성하겠습니다.
이상한 답변을 왜 할까요?

현상

피하고 싶은 표현이 다시 나오고, 수정할수록 말투 기준이 흐려집니다.

문제와 설정 위치

  • 계속 피해야 하는 표현 → Instruction: “변명처럼 들리는 표현은 쓰지 마.”
  • 이번 답변에서 중요 → Prompt: “배송 지연 안내문이고, 변명처럼 들리지 않게 써줘.”
  • 마지막 확인 필요 → 검수 요청: “변명처럼 들리는 표현이 있는지 점검해줘.”

계속 피해야 하는 표현은 저장하고, 이번 답변에서 중요한 조건은 요청 앞에 다시 적습니다.

고쳐야 할 부분

  • 계속 피해야 하는 표현은 Instruction에 저장합니다.
  • 중요한 조건은 이번 요청 맨 앞에 다시 적습니다.
  • 마지막에 “변명처럼 들리는 표현이 있는지 확인해줘”를 붙입니다.

“분명히 말했는데 또 까먹었어요.”

AI의 기억은 사람의 기억처럼 무제한으로 보존되는 방식이 아닙니다
나고객 안내문은 3문장 이하로, 정중하게 써줘.
AI안녕하세요 고객님. 먼저 이용에 불편을 드려 죄송합니다. 현재 물류센터 사정으로 인해...
나3문장 이하라고 했잖아.
AI조건을 다시 반영해 3문장으로 줄이겠습니다.
이상한 답변을 왜 할까요?

현상

앞에서 정한 길이와 말투 조건이 약해지고, 긴 대화일수록 조건이 섞입니다.

문제와 설정 위치

  • 3문장 조건을 잊음 → Prompt: “3문장 이하로 작성해줘.”
  • 정중한 말투가 반복 필요 → Memory: “고객 안내문은 항상 정중하고 짧게 작성해줘.”
  • 대화가 길어짐 → 새 요청: “조건 유지: 3문장 이하, 정중한 말투.”

계속 유지할 기준은 Memory에 저장하고, 이번 답변의 필수 조건은 Prompt에 다시 적습니다.

고쳐야 할 부분

  • 중요한 기준은 새 요청 맨 앞에 다시 적습니다.
  • “3문장 이하, 정중한 말투”처럼 짧게 고정합니다.
  • 반복 선호는 Memory에, 필수 조건은 요청에 넣습니다.

“틀렸는데 너무 자신 있게 말해요.”

AI는 정답 창고에서 꺼내기보다, 문맥상 자연스러운 답을 만들어냅니다
나Codex 최신 설정 방법을 알려줘.
AI최신 메뉴는 설정의 개발자 탭에 있습니다.
나그거 확실해? 화면이랑 다른데?
AI확인된 근거 없이 추측했습니다. 확인 필요로 표시하겠습니다.
이상한 답변을 왜 할까요?

현상

근거가 없는데 단정하고, 최신 정보나 숫자를 그럴듯하게 말합니다.

문제와 설정 위치

  • 근거 없이 단정함 → Instruction: “확인 못 한 내용은 추측이라고 표시해.”
  • 최신 정보가 필요함 → Prompt: “최신 정보인지 확인하고, 모르면 모른다고 말해줘.”
  • 반복 확인 필요 → Skill: “근거 확인 → 추측 분리 → 체크리스트 순서로 검토해.”

확인 기준은 Instruction에 저장하고, 반복 검증은 Skill로 절차화합니다.

고쳐야 할 부분

  • “확인 못 한 내용은 추측이라고 표시해”를 지침에 넣습니다.
  • 근거와 추측을 분리해서 쓰게 합니다.
  • 마지막에 확인 체크리스트를 실행하게 합니다.
PART 3

역할을 나눠 일하는 AI

Main Agent는 전체 요청과 기준을 보고, 필요한 Skill과 Sub Agent를 골라 결과물을 완성합니다.

5컷으로 보는 AI 작업 흐름

요청 이해 → 참고 자료 확인 → 절차와 역할 선택 → 위임 → 최종 정리

1-2단계: 요청과 참고 자료를 확인한다

AI는 “지금 참고할 수 있는 정보”를 바탕으로 답을 만듭니다

3-4단계: 절차와 역할을 고르고 일을 맡긴다

반복 절차와 역할이 분리되면 반복 작업이 안정됩니다

5단계: 결과를 검토하고 전달한다

Sub Agent의 결과물을 그대로 내지 않고, Main Agent가 목표에 맞게 마무리합니다
PART 4

내 업무 방식을 저장하기

AI가 내 기준대로 일하려면 기준과 반복 절차를 함께 이해해야 합니다. 기준, 절차, 역할, 도구 패키지를 저장해 첫 결과를 점점 나에게 가깝게 만듭니다.

Skill은 반복 업무 절차입니다

짧은 요청만 해도 같은 순서와 기준으로 일하게 만드는 .md 절차서입니다

언제 Skill로 만들까

  • 매번 같은 순서로 만드는 요리.
  • 도구를 쓰는 순서가 중요한 요리.
  • 마지막 맛 확인 기준이 정해진 요리.
예시 요청
/tomato-pasta
2인분, 마늘은 적게, 20분 안에 만들어줘.
# SKILL.md 예시 --- name: tomato-pasta description: 해야 할 것과 하지 말아야 할 것을 지켜 토마토 파스타 만들기 --- 1. 주문 확인 - 손님 수, 금지 재료, 매운맛 정도를 확인한다. - 빠진 정보가 있으면 추측하지 말고 총 주방장 Agent에게 묻는다. 2. 도구 준비 - 냄비, 팬, 집게, 체, 접시를 순서대로 꺼낸다. - 면 삶을 물을 먼저 올리고, 팬은 소스 직전에 예열한다. 3. 재료 준비 - 마늘, 토마토소스, 면, 소금, 올리브오일을 확인한다. - 금지 재료가 있으면 바로 제외한다. 4. 조리 순서 - 면을 삶는 동안 팬에서 소스를 만든다. - 면수 한 국자를 넣고 농도를 맞춘 뒤 면을 버무린다. 5. 최종 확인 - 간, 면 익힘, 소스 농도, 플레이팅을 확인한다. - 결과와 조정할 점을 총 주방장 Agent에게 보고한다.

Agent는 역할을 맡은 AI입니다

AGENT.md에 역할, 맡길 일, 판단 기준을 적어 역할별로 일을 나눕니다

재료 손질 담당 Agent

조리에 바로 쓸 수 있도록 재료를 준비합니다.

# AGENT.md # 재료 손질 담당 Agent ## 역할 - 조리 전 재료를 깨끗하게 준비한다. - 상한 재료와 알레르기 위험을 먼저 보고한다. ## 해야 할 일 1. 재료를 씻고 물기를 제거한다. 2. 껍질, 씨, 힘줄처럼 먹지 않는 부분을 뺀다. 3. 레시피에 맞는 크기로 썰어 담는다. ## 결과 - 손질 완료 재료 - 교체 필요 재료 목록 - 다음 담당에게 줄 주의사항

양식 담당 Agent

손질된 재료로 양식 메뉴를 조리합니다.

# AGENT.md # 양식 담당 Agent ## 역할 - 파스타, 스테이크 같은 양식 메뉴를 맡는다. - 맛과 플레이팅을 함께 확인한다. ## 해야 할 일 1. 전달받은 재료와 레시피를 확인한다. 2. 굽기, 삶기, 소스 만들기 순서로 조리한다. 3. 접시에 보기 좋게 담는다. ## 결과 - 완성된 양식 메뉴 - 간, 굽기, 소스 상태 메모 - 부족한 재료나 시간 이슈

찜 요리 담당 Agent

온도와 시간을 지키며 찜 요리를 완성합니다.

# AGENT.md # 찜 요리 담당 Agent ## 역할 - 찜기에 넣는 순서와 익힘 시간을 관리한다. - 재료가 마르지 않게 수분과 불 세기를 본다. ## 해야 할 일 1. 단단한 재료부터 넣는다. 2. 뚜껑을 열기 전 시간을 확인한다. 3. 익힘 상태와 간을 마지막에 점검한다. ## 결과 - 완성된 찜 요리 - 익힘 상태 메모 - 다음 조리 때 조정할 시간

처음부터 내 기준대로 일하지는 않습니다

반복 경험을 Instruction, Memory, Skill, Agent로 저장하면 첫 결과가 점점 나에게 가까워집니다
1

첫 요청

결과가 너무 일반적이거나 방향이 빗나갈 수 있습니다.

2

핵심 3줄

AI와 친해지는 첫걸음은 내 업무의 해야 할 것과 하지 말아야 할 것을 분명히 적는 것입니다.
Instruction, Memory, Skill, Agent는 내 기준을 반복 가능하게 만드는 장치입니다.
작게 저장하고 반복해서 고칠수록 첫 결과가 점점 나에게 가까워집니다.

완벽한 복제본이 아니라
“내 기준으로 일하는 방식”을 만듭니다

오늘 한 작업들은 AI가 나의 기준, 선호, 절차, 역할표를 참고해
처음부터 조금 더 나답게 일하도록 만드는 과정입니다.

정리하면
좋은 프롬프트는 이번 작업의 기준표이고, Instruction / Memory / Skill / Agent는 AI가 내 기준대로 일하게 만드는 구성 요소입니다.

Q & A

“내가 자주 반복해서 시키는 일” 하나를 떠올려보세요.
그 일의 해야 할 것, 하지 말아야 할 것, 반복 절차, 필요한 역할은 무엇일까요?

[표지] 제목과 부제를 읽습니다. 사람을 복제한다는 뜻이 아니라, 내 기준과 일하는 방식을 AI가 더 잘 참고하게 만드는 과정이라고 먼저 정리합니다.
[목차] 좋은 프롬프트에서 시작해, Instruction, Context, Memory, Skill, Agent를 이해하고, 마지막에는 Codex에서 실제로 지침과 Memory를 설정하는 흐름이라고 안내합니다.
[오프닝] AI와 친해진다는 말을 “내가 원하는 일의 기준을 함께 정하는 것”으로 설명합니다. 오늘은 기술 자체보다 기준을 전달하고 저장하는 법에 집중한다고 말합니다.
[퀴즈] 바로 정답을 말하지 말고 수강생에게 묻습니다. 좋은 프롬프트에는 어떤 정보가 있어야 할지 자유롭게 답을 받아본 뒤 다음 장으로 넘어갑니다.
[좋은 프롬프트란] 왼쪽과 오른쪽은 같은 요청입니다. 차이는 역할, 목표, 맥락, 해야 할 것 / 하지 말아야 할 것, 출력 형식, 확인 기준이 분리되어 있는지입니다. 수강생에게 문장 길이가 아니라 판단 기준의 유무를 보게 합니다.
[개념 파트 전환] 좋은 프롬프트의 항목들이 실제 AI 도구에서는 지침, 기억, 절차, 역할 같은 형태로 저장될 수 있다고 연결합니다.
[Instruction] Instruction은 총 주방장이 항상 지켜야 하는 운영 원칙입니다. 손님 요청 확인, 담당 Agent에게 일을 나누는 기준, 하지 말아야 할 금지선, 마지막 검수 방식을 파일로 남긴다고 설명합니다.
[Memory] Memory는 총 주방장이 계속 참고하면 좋은 반복 기억입니다. 단골 손님 취향, 자주 쓰는 재료, 선호하는 설명 방식처럼 참고하면 결과가 좋아지는 정보를 남긴다고 설명합니다. 다만 알레르기와 금지 재료처럼 반드시 지켜야 하는 조건은 Memory에만 맡기지 말고 매번 요청에서 다시 확인해야 합니다.
[Context] Context는 AI와 주고받은 질문, 답변, 수정 요청, 첨부 자료가 이어진 대화의 맥락입니다. AI에게 질문하고, 답을 받고, 다시 고쳐달라고 말하는 그 흐름 전체라고 설명합니다.
[Context Window] 막대 전체가 AI가 한 번에 참고할 수 있는 범위입니다. 대화가 길어지면 이전 내용은 압축되고, 세부 표현이나 작은 조건 같은 데이터 일부는 유실될 수 있습니다. 중요한 기준을 다시 적으면 새 요청 쪽 비중이 커진다고 설명합니다.
[Skill] Skill은 반복 업무 절차입니다. 특정 요리를 만들 때 재료 확인, 도구 준비, 조리 순서, 마지막 맛 확인이 정해져 있는 것처럼 반복되는 순서를 저장한다고 설명합니다.
[Agent] Agent는 책임을 가진 역할입니다. 화면의 예시는 총 주방장 Agent로, 손님 요청을 확인하고 재료 손질, 양식, 찜 요리 담당 Agent에게 일을 나눈 뒤 최종 결과를 검토합니다. Skill이 절차라면 Agent는 그 절차 안에서 판단하고 위임하는 역할이라고 설명합니다.
[한 줄 정리] 표를 천천히 읽습니다. Prompt, Instruction, Context, Context Window, Memory, Skill, Agent가 서로 다른 역할을 한다는 점만 남기면 됩니다.
[사례 파트 전환] 이제 개념을 실제 불편함에 연결합니다. AI가 이상하게 군다고 느끼는 순간은 대개 요청, 기준, 기억, 확인 방식 중 하나가 흐려졌기 때문입니다.
[말 안 들어요] 왼쪽 대화에서 금지 표현이 다시 나오는 장면을 읽습니다. 오른쪽에서는 현상, 문제와 설정 위치, 고쳐야 할 부분 순서로 설명합니다. 핵심은 계속 피해야 하는 표현은 Instruction에 저장하고, 이번 답변에서 중요한 조건은 Prompt에도 다시 써야 한다는 점입니다.
[까먹어요] 왼쪽 대화에서 비개발자 대상 조건이 약해지는 장면을 읽습니다. 오른쪽에서는 Context, Context Window, Memory를 연결해 설명합니다. 대화가 길어지면 일부 조건이 압축되거나 유실될 수 있으므로 중요한 기준은 새 요청 맨 앞에 다시 적어야 합니다.
[자신 있게 틀려요] 왼쪽 대화에서 확인되지 않은 정보를 단정하는 장면을 읽습니다. 오른쪽에서는 Context, Instruction, Skill을 연결합니다. 근거 자료가 없으면 추측이 섞일 수 있으므로 모르면 모른다고 말하는 지침과 확인 체크리스트가 필요합니다.
[동작 흐름 전환] Main Agent가 전체 요청과 기준을 보고 필요한 절차와 역할을 골라 결과물을 완성한다는 큰 흐름만 설명합니다.
[5컷 흐름] 요청 이해, 참고 자료 확인, Skill과 Sub Agent 선택, 위임, 최종 정리 순서로 그림을 따라갑니다. 앞에서 배운 개념과 연결해줍니다.
[1-2단계] AI가 요청만 보는 것이 아니라 지침, 기억, 파일, 이전 대화까지 함께 참고하려고 한다고 설명합니다. 현재 참고 범위 안의 정보가 더 강하게 영향을 줍니다.
[3-4단계] Skill은 절차이고 Sub Agent는 역할입니다. 어떤 순서로 만들지와 누가 어떤 관점으로 볼지를 분리하면 반복 작업이 안정됩니다.
[5단계] Sub Agent가 만든 결과는 초안입니다. Main Agent가 원래 요청, 기준, 대상, 출력 형식을 다시 보고 최종 정리합니다.
[활용 파트 전환] 이제 내 업무 방식을 저장하는 방법으로 넘어갑니다. 목표는 매번 긴 프롬프트를 쓰는 것이 아니라 기준, 절차, 역할표를 저장하는 것입니다.
[Skill 상세] 실제 SKILL.md처럼 번호가 붙은 절차를 읽습니다. 토마토 파스타 예시에서는 주문 확인, 도구 준비, 재료 준비, 조리 순서, 최종 확인이 반복 업무의 뼈대가 됩니다.
[Agent 상세] 역할을 배정하는 것이 Agent의 핵심입니다. 화면의 .md 예시는 실제 요리 담당 역할처럼 작성되어 있습니다. 역할, 해야 할 일, 결과물이 맞아야 Main Agent가 정확히 위임할 수 있다고 설명합니다.
[반복 개선] 처음부터 내 기준대로 일하게 만들 수는 없습니다. 너무 어렵다, 근거가 부족하다, 예시가 약하다 같은 피드백을 지침, 기억, 절차로 옮길수록 다음 결과가 좋아집니다.
[체크리스트] 수강생이 오늘 바로 할 수 있는 네 가지를 정리합니다. 해야 할 것 / 하지 말아야 할 것 쓰기, 운영 원칙 저장하기, 절차를 Skill화하기, 역할 나누기입니다.
[핵심 3줄] 세 문장을 천천히 읽습니다. AI와 친해지는 첫걸음은 해야 할 것과 하지 말아야 할 것이고, Instruction, Memory, Skill, Agent는 그 기준을 반복 가능하게 만드는 장치입니다.
[내 기준대로 일하는 AI] 완벽한 사람 복제가 아니라 내 기준으로 일하는 방식을 만드는 것이라고 정리합니다.
[개인 맞춤 지침] 스크린샷의 번호를 따라 설명합니다. 1번은 개인 맞춤 설정 메뉴, 2번은 맞춤형 지침 입력칸, 3번은 내 정보 입력 영역입니다. 맞춤형 지침에는 해야 할 것, 하지 말아야 할 것, 말투, 출력 형식을 넣습니다.
[Memory 설정] 스크린샷의 번호를 따라 설명합니다. 메모리 영역에서 저장된 메모리 참고를 켜고, 관리 버튼으로 저장된 내용을 확인하거나 삭제합니다. 새 기억은 설정 화면 입력칸이 아니라 대화창에서 “기억해줘: ...”라고 말해 저장한다고 설명합니다.
[가능성] Codex와 Claude Code 같은 제작형 AI는 문서, 작은 웹 결과물, 자동화, 검토와 개선까지 함께할 수 있습니다. 답변 창을 넘어 내 기준으로 함께 만드는 작업자라고 정리합니다.
[Figma 플러그인 제작 사례] 먼저 요구사항을 읽습니다. 피그마 화면의 텍스트를 한꺼번에 모아 다국어 key를 설정하고, 적용된 key를 annotation으로 확인하는 것이 목표였다고 설명합니다. 그 다음 Instruction과 Memory로 작업 기준을 세팅하고, Brainstorm과 Deep Interview Skill로 요구사항을 개발 기획서로 바꿨다고 말합니다. 이후 “이 기획서를 기준으로 Figma Plugin을 만들어줘”라고 구현을 맡겼고, 우측 영상은 그 결과물이라고 연결합니다.
[Q&A] 질문을 받습니다. 질문이 없으면 “내가 자주 반복해서 시키는 일 하나를 떠올려보세요. 그 일의 해야 할 것, 하지 말아야 할 것, 반복 절차, 필요한 역할은 무엇일까요?”라고 유도합니다.
