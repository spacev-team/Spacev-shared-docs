1. AI가 말을 안 들어요

“하지 말라고 했는데
자꾸 계속해요.”

이건 AI가 반항하는 게 아니라,
기준이 충분히 고정되지 않은 경우가 많아요.

Prompt(요청)

Prompt(요청)는
오늘 한 번 부탁하는 말이에요.

부탁 이 회의 내용을 요약해줘
한계 매번 새로 말해야 해요

프롬프트는 즉석 요청이에요. 지금 할 일은 말할 수 있지만, 내 기본 기준까지 계속 기억하게 만들지는 못해요.

Instruction(지침)

Instruction(지침)은
반복해서 지킬 기준이에요.

응답은 “~해요”로 써요.

먼저 결론을 말하고, 그다음 이유를 붙여요.

확실하지 않은 내용은 “확인 필요”라고 표시해요.

매번 같은 잔소리를 하게 된다면, 그건 프롬프트가 아니라 지침에 넣을 내용이에요.

한 줄 정리

Prompt(요청)는 오늘의 부탁,
Instruction(지침)은 매번의 기준이에요.

Prompt(요청)

이번 답변에 필요한 요청

Instruction(지침)

계속 지켜야 하는 답변 방식

2. AI가 자꾸 까먹어요

“분명히 말했잖아요.
왜 또 물어봐요?”

AI의 기억은 사람처럼 오래 보관되는 기억과,
지금 답변을 위한 작업 기억으로 나뉘어요.

Context(작업 기억)

Context(작업 기억)는 AI가 지금 보고 있는
대화의 묶음이에요.

컨텍스트

AI가 이번 답변을 만들 때 참고하는 대화, 지침, 파일, 예시 같은 기억의 묶음이에요.

1. 압축 전 대화 기록과 메모리가 같이 들어와요

질문했던 내용과 기억해둔 짧은 문장이 이번 답변의 재료가 돼요.

대화 기록 “비개발자도 이해하게 설명해줘” “Before/After 예시를 넣어줘” “말투는 너무 딱딱하지 않게 해줘”
기억된 정보 교육자료를 자주 만들어요 “~해요” 말투를 선호해요
여유
2. 대화 압축 후 요약본만 남고 여유가 생겨요

대화가 길어지면 일부 기억은 빠지거나 약해질 수 있어요.

압축된 요약 일부 기억 유실 가능 여유 컨텍스트
3. 다음 대화 압축본 뒤에 새 요청이 다시 쌓여요

그래서 중요한 기준은 새 요청 앞에 다시 올려주는 편이 좋아요.

압축본 새 요청 추가 조건 여유

Memory(기억)

Memory(기억)는 AI가 나에 대해
기억해도 되는 정보예요.

나는 비개발자 대상 교육자료를 자주 만들어요.

답변은 한국어로, 너무 딱딱하지 않게 받아보고 싶어요.

표나 체크리스트를 선호해요.

메모리는 “내가 어떤 사람인지”, “내가 자주 원하는 방식이 무엇인지”에 가깝습니다.

한 줄 정리

Context(작업 기억)는 지금 대화,
Memory(기억)는 나에 대한 반복 정보예요.

Context(작업 기억)

이번 답변을 만들 때 참고하는 작업 기억

Memory(기억)

대화가 바뀌어도 참고하면 좋은 사용자 정보

3. AI가 그럴듯하게 틀려요

“틀렸는데
너무 자신 있게 말해요.”

AI는 사실 창고에서 정답을 꺼내기보다,
문맥상 자연스러운 답을 만들어내요.

정확도가 필요할 때

답을 요구하기 전에
확인 방식을 정해요.

모르면 모른다고 말하기 근거와 추측을 구분하기 확인할 항목을 체크리스트로 만들기

정확한 정보가 필요할 때는 “답해줘”보다 “어떻게 확인했는지 보여줘”가 더 중요해요.

4. 결과가 너무 평범해요

“누구나 할 법한 말만
나와요.”

매번 같은 일을 한다면,
한 번의 요청보다 작업 방식 자체를 저장하는 편이 좋아요.

Skill(작업 템플릿)

Skill(작업 템플릿)은 반복 작업을 위한
저장된 진행 방식이에요.

언제 쓰는지 어떤 순서로 하는지 어떤 형식으로 내는지

스킬은 “이 일을 할 때는 항상 이 순서로 해줘”를 저장해두는 개념이에요.

Instruction(지침)과 Skill(작업 템플릿)

지침은 기본 습관,
스킬은 특정 작업 방식이에요.

Instruction(지침)

답변 말투, 금지 조건, 확인 기준

Skill(작업 템플릿)

회의록 정리, 공지 작성, 리서치 요약 같은 반복 작업

5. AI가 답만 주고 끝나요

“좋은 말은 해주는데
작업은 결국 내가 해요.”

챗봇은 답을 주는 화면이고,
Agent(작업자 AI)는 일을 맡길 수 있는 방식에 가까워요.

Agent(작업자 AI)

Agent(작업자 AI)는 역할을 맡고
도구를 써서 일을 진행하는 AI예요.

역할을 이해해요 파일과 도구를 다뤄요 결과물을 확인해요

비유하면 “질문에 답하는 사람”에서 “작업을 맡아 진행하는 사람”으로 넘어가는 거예요.

챗봇과 Agent(작업자 AI)

챗봇은 답을 주고,
Agent(작업자 AI)는 결과물을 만져요.
