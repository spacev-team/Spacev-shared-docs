03
각 스킬 기능
/growth
오케스트레이터 — 목표를 읽고 스킬을 골라 실행한 뒤 종합
├── Step 1. 사용자 목표·목적 파악
├── Step 2. 자동 분류
│ ├── DATA → "왜?", 데이터 파일, 패턴 질문
│ ├── STRATEGY → "뭘 해야 해?", 기획 방향, 가설
│ └── PIPELINE → "분석하고 기획까지", 둘 다 필요
├── Step 3. 알맞은 스킬 실행 (자동)
└── Step 4. 결과 종합 → 데이터 / 전략 / 다음 액션
/data-trace
DB 스캔 → 인사이트 랭킹 → 원인 추적
├── Phase 1. Scan
│ ├── 데이터 품질 점검 (중복·결측·날짜 이상)
│ └── 세그먼트·상태값 분포 파악 (원본 파일 수정 안 함)
├── Phase 2. Insight Mining
│ ├── Surprise × Leverage 점수 계산
│ └── 상위 인사이트 3~7개 랭킹
└── Phase 3. Trace
    ├── 레인 A: 행동/동기 원인
    ├── 레인 B: 구조/UX 원인
    ├── 레인 C: 데이터 품질 원인 (먼저 의심)
    └── 반론 라운드 → 핵심 미지수 → 다음 탐색
/strategy-interview
소크라테스 인터뷰 → 모호성 게이트 → 전략 브리프
├── Round 0. 전략 토폴로지 확인 (문제 구성 요소 잠금)
├── Round 1~. 1문 1답 소크라테스
│ ├── 가장 불명확한 차원 타깃 → 한 가지만 질문
│ └── 답변 후 모호성 점수 계산 (0~100%)
├── 챌린지 모드
│ ├── Round 4: 가정 도전 (이게 정말 맞는 전제인가?)
│ ├── Round 6: 레버리지 탐색 (하나만 바꾼다면?)
│ └── Round 8: 근본원인 추적 (증상 vs 원인)
└── 모호성 25% 미만 달성 → 전략 브리프 생성
04
스킬이 연결되는 방식 (PIPELINE 모드)
사용자: "분석하고 기획까지 잡아줘"
/growth 목표 입력 받음
└── PIPELINE 판단
    ├── /data-trace 실행
    │ ├── Scan → Insight Mining → Trace
    │ └── 결과: 인사이트 + 원인 가설 + 핵심 미지수
    │
    ├── 컨텍스트 자동 주입 ↓
    │ └── 확인된 사실 전달, 재질문 없음
    │
    ├── /strategy-interview 실행
    │ └── 미결 항목만 질문 → 모호성 25% 미만 → 브리프
    │
    └── 종합 출력 데이터 → 전략 → 다음 액션
컨텍스트 자동 주입이란? data-trace에서 확인된 인사이트와 원인 가설이 strategy-interview에 자동으로 전달됩니다. 이미 아는 사실은 다시 묻지 않고, 아직 불명확한 것(성공 기준, 개입 방식)에만 집중해서 질문합니다.
05
사용법
📊 데이터 분석이 필요할 때
/growth
이 CSV에서 인사이트를 찾고 싶다
├── DATA로 분류
├── /data-trace 실행
└── 인사이트 + 원인 추적
🗺️ 전략 방향이 필요할 때
/growth
보류율을 낮춰 공개방 수를 늘리고 싶다
├── STRATEGY로 분류
├── /strategy-interview 실행
└── 인터뷰 → 전략 브리프
⚡ 분석 + 기획 한 번에
/growth
퍼널 분석하고 다음 액션까지 잡아줘
├── PIPELINE으로 분류
├── data-trace → strategy-interview
└── 결과 종합 + 다음 액션
스킬 단독 실행도 가능
├── /data-trace 보류사유별_분석결과.csv
└── /strategy-interview 재운영 임대인을 다시 활성화하고 싶다
