About

플러그인이란?

PLUGIN

AI에 기능을 꽂아 쓰는 연결 묶음입니다

AI 자체를 바꾸는 것이 아니라, 필요한 도구와 절차를 플러그처럼 연결해 더 많은 일을 맡기는 방식입니다.

MCPNotion, Figma, Slack 같은 외부 서비스와 연결합니다.
Skill반복되는 업무 절차와 검토 기준을 불러옵니다.
Agent자료 정리, 검토, 제작처럼 역할을 나눠 맡깁니다.
Tool문서, 파일, 브라우저 같은 실제 작업 기능을 실행합니다.

핵심: AI에게 기능 플러그를 꽂아, 답변을 넘어 실제 작업까지 맡깁니다.

기능 묶음을 바꿔 꽂으면 “회의 정리 플러그인”, “디자인 검토 플러그인”처럼 다른 일을 맡길 수 있습니다. 그럼 이런 묶음은 어떻게 직접 만들까요? → 아래 03. 직접 만들기에서 단계별로 따라가 봅니다.
Why

왜 플러그인을 사용하는가?

한 줄로 — 잘 만들어 둔 자동화 묶음을, 나 혼자만 쓰지 말고 팀·조직 전체가 쉽게 쓰게 하기 위해서입니다.

설정은 그냥 내 폴더(.claude/)에 둬도 동작합니다. 그런데도 굳이 플러그인으로 만드는 이유는 공유·재사용·버전 관리 때문입니다.

그냥 설정 (.claude/)

  • 적용 범위 · 내 프로젝트 한 곳에서만
  • 공유 · 파일을 일일이 복사해서 전달
  • 버전 · 누가 어떤 버전 쓰는지 모름
  • 표준화 · 사람마다 설정이 제각각

플러그인

  • 적용 범위 · 여러 프로젝트에서 재사용
  • 공유 · /plugin install 한 번이면 끝
  • 버전 · 버전을 올리면 팀원 전체가 동기화
  • 표준화 · 팀 베스트 프랙티스를 코드처럼 배포
1

재사용

한 번 잘 만들어 두면 모든 프로젝트에서 똑같이 동작합니다.

2

공유

마켓플레이스(사내 비공개 저장소도 가능)에 올려두면, 팀원은 명령어 한 줄로 설치합니다. “내 노하우 세팅”이 곧 “팀 자산”이 됩니다.

3

버전 관리 · 표준화

규칙을 업데이트하면 모두가 같은 최신 버전을 사용합니다. 팀 전체가 같은 품질·같은 방식으로 작업합니다.

Build

플러그인 직접 만들기 (Step by Step)

예시로 인사를 건네는 hello 스킬을 가진 my-first-plugin을 직접 만들어 봅니다. 앞에서 본 개념 단위(MCP·Skill·Agent·Tool)는 실제로는 .mcp.json · skills/ · agents/ · hooks/ 폴더와 신분증 역할의 plugin.json으로 포장됩니다.
★ Best Practice

보통은 Claude에게 통째로 시킵니다

파일을 하나하나 손으로 만들기보다, Claude Code에게 “이런 플러그인 만들어줘”라고 시키는 게 가장 빠릅니다. 아래 Step들은 그렇게 만들어진 결과물의 구조를 이해하고 검토하기 위한 것입니다. 

team-review 라는 플러그인을 만들어줘.

목표: 우리 팀의 코드 리뷰를 자동화하는 거야.

포함할 것:
1) 우리 팀 컨벤션을 점검하는 convention-check 스킬
2) PR을 리뷰하는 code-reviewer 에이전트 (model은 sonnet)
3) Slack 연동 MCP 서버 (서버 링크는 내가 전달할게)

.claude-plugin/plugin.json 까지 갖춰서 폴더 구조로 만들고,
마지막에 `claude plugin validate` 로 검증까지 해줘.
  • 목표·포함 요소·폴더 구조를 구체적으로 적습니다 (위처럼 번호로 나열).
  • 한 번에 다 맡기기보다 스킬 → 에이전트 → MCP 순으로 하나씩 확인하며 진행합니다.
  • MCP는 서버 링크를 함께 전달하면 Claude가 .mcp.json을 정확히 만들어 줍니다.
  • 검증(claude plugin validate)·로컬 테스트(--plugin-dir)까지 함께 요청합니다.
  • 완성 후 README.md와 버전(version)도 챙기게 합니다.
Step 0

사전 준비

  • Claude Code가 설치·로그인되어 있어야 합니다.
  • /plugin 명령이 보이지 않으면 최신 버전으로 업데이트하세요.
Claude Code 안에서 
/plugin   # 명령이 보이면 준비 완료
Step 1

플러그인 폴더 만들기

  • 플러그인은 자기만의 폴더 하나로 시작합니다.
  • 이 폴더 안에 스킬·에이전트·설정이 모두 담깁니다.
terminal 
mkdir my-first-plugin
Step 2

Manifest(신분증) 만들기

  • name · 고유 이름이자 명령어 네임스페이스 (/my-first-plugin:hello)
  • description · 플러그인 관리자에서 보이는 설명
  • version · 올리면 사용자에게 업데이트가 전달됨
  • author · (선택) 작성자 표기
terminal 
mkdir my-first-plugin/.claude-plugin
my-first-plugin/.claude-plugin/plugin.json 
{
  "name": "my-first-plugin",
  "description": "기본기를 익히는 인사 플러그인",
  "version": "1.0.0",
  "author": { "name": "당신 이름" }
}
⚠️ 자주 하는 실수 — skills/, agents/, hooks/는 .claude-plugin/ 안에 넣으면 안 됩니다. .claude-plugin/ 안에는 plugin.json만 두고, 나머지 폴더는 모두 플러그인 루트에 둡니다.
Step 3

Skill(스킬) 추가하기

  • 스킬은 skills/<이름>/SKILL.md 형태로 만듭니다.
  • 폴더 이름(hello)이 곧 명령어 이름이 됩니다.
  • description을 보고 Claude가 알아서 호출하기도 합니다.
terminal 
mkdir -p my-first-plugin/skills/hello
my-first-plugin/skills/hello/SKILL.md 
---
description: 사용자에게 친근하게 인사하기
---

사용자에게 따뜻하게 인사하고, 오늘 무엇을 도와줄지 물어봅니다.
📁 인식 위치 — 스킬은 플러그인 루트의 skills/<이름>/SKILL.md 위치여야 인식됩니다. (.claude-plugin/ 안에 넣으면 ❌)
스킬이 인식되는 위치 (예시) 
my-first-plugin/
├── .claude-plugin/
│   └── plugin.json
└── skills/
    └── hello/
        └── SKILL.md          # → /my-first-plugin:hello
인기 스킬 (claudemarketplaces.com) — 클릭하면 스킬 상세로 이동
pdf PDF 병합·분할·텍스트/표 추출, OCR xlsx 엑셀 데이터·수식·차트 생성·편집 docx Word 문서 생성·편집·서식 pptx 프레젠테이션 슬라이드 제작
Step 4

Agent(에이전트) 추가하기

  • 에이전트는 agents/ 폴더에 마크다운 파일 1개로 만듭니다.
  • frontmatter에 name·description·model, 본문에 역할(시스템 프롬프트)을 적습니다.
  • description을 보고 Claude가 자동 호출하거나, /agents에서 직접 부릅니다.
terminal 
mkdir my-first-plugin/agents
my-first-plugin/agents/code-reviewer.md 
---
name: code-reviewer
description: PR과 변경된 코드를 리뷰합니다. 코드 리뷰나 PR 점검이 필요할 때 사용하세요.
model: sonnet
---

당신은 시니어 코드 리뷰어입니다. 다음을 점검하세요:
1. 버그와 엣지 케이스
2. 보안 문제
3. 팀 코드 컨벤션 준수
4. 테스트 커버리지
📁 인식 위치 — 에이전트는 플러그인 루트의 agents/<이름>.md 위치여야 /agents에 인식됩니다. (.claude-plugin/ 안에 넣으면 ❌)
에이전트가 인식되는 위치 (예시) 
my-first-plugin/
├── .claude-plugin/
│   └── plugin.json
└── agents/
    └── code-reviewer.md      # → /agents 에 등록
인기 에이전트 (wshobson/agents) — 클릭하면 .md 파일로 이동
backend-architect 백엔드 아키텍처·API 설계를 잡아주는 에이전트 frontend-developer 프론트엔드 UI·컴포넌트 개발 전문 security-auditor 보안 취약점·인증 흐름 점검 test-automator 테스트 작성·커버리지 자동화
Step 5

MCP(외부 도구) 연결하기

  • MCP는 Slack·Figma·Notion 같은 외부 서비스를 연결합니다. 종류가 매우 다양합니다.
  • .mcp.json을 손으로 쓰기보다, 보통 MCP 서버 링크를 Claude에게 전달해 “추가해줘”라고 시킵니다 — Claude가 .mcp.json을 알맞게 만들어 줍니다.
  • 플러그인이 켜지면 서버가 자동 시작되어 Claude의 도구로 쓰입니다.
Claude Code에게 요청 
이 MCP 서버를 my-first-plugin 플러그인에 추가해줘:
https://github.com/modelcontextprotocol/servers (Slack)

.mcp.json 에 알맞게 설정하고,
필요한 환경변수(SLACK_BOT_TOKEN 등)도 함께 정리해줘.
💡 MCP는 종류가 다양합니다 — 링크만 바꿔 전달하면 어떤 서비스든 같은 방식으로 연결됩니다.
Slack메시지 전송·채널 조회
Figma디자인 파일·컴포넌트 읽기
Step 6

로컬에서 테스트하기

  • --plugin-dir 플래그로 설치 없이 바로 불러옵니다.
  • 보통 플러그인 폴더가 아니라, 플러그인을 실제로 사용할 다른 경로(테스트용 프로젝트)에서 Claude를 켜고 --plugin-dir로 플러그인 위치를 가리킵니다 — 진짜 설치된 것처럼 동작하는지 보기 위해서입니다.
  • 스킬·에이전트가 등록됐는지 확인하고, 수정할 때마다 /reload-plugins로 다시 불러옵니다.
terminal — 플러그인을 사용할 다른 경로에서 
cd ~/work/test-project        # 플러그인 폴더가 아닌 다른 경로
claude --plugin-dir ~/dev/my-first-plugin
Claude Code 안에서 
/my-first-plugin:hello   # 스킬 동작 확인
/agents                  # 에이전트 등록 확인
💡 /my-first-plugin:hello처럼 앞에 플러그인이름:이 붙는 이유 — 여러 플러그인에 같은 이름의 스킬(hello)이 있어도 충돌하지 않도록, 플러그인 이름(plugin.json의 name)을 네임스페이스로 붙입니다.
Step 7

팀에 공유하기 (배포)

  • 팀이 /plugin install 한 줄로 받게 하려면 마켓플레이스를 만듭니다.
  • 마켓플레이스는 플러그인 목록을 담은 git 저장소이고, 사내 비공개(private) 저장소도 그대로 됩니다.
  • 배포 전 claude plugin validate로 검증하고, README.md를 추가합니다.
  • 플러그인과 marketplace.json을 이미 만들어 둔 git 저장소에 올리면 배포 완료입니다 — 이 작업도 Claude에게 요청하면 됩니다.
  • 여기서 02. 왜 쓰는가의 “공유·버전관리·표준화” 효과가 실현됩니다.
1) 검증 — terminal 
claude plugin validate ./my-first-plugin
2) 마켓플레이스 저장소 구조 
team-plugins/                        # git 저장소
├── .claude-plugin/
│   └── marketplace.json
└── my-first-plugin/                 # Step 1~5에서 만든 플러그인
    ├── .claude-plugin/plugin.json
    ├── skills/
    ├── agents/
    └── .mcp.json
team-plugins/.claude-plugin/marketplace.json 
{
  "name": "spacev-team",
  "owner": { "name": "SpaceV", "email": "teddy@spacev.kr" },
  "plugins": [
    {
      "name": "my-first-plugin",
      "source": "./my-first-plugin",
      "description": "기본기를 익히는 인사 플러그인",
      "version": "1.0.0"
    }
  ]
}
3) 원격 저장소에 올리기 — Claude Code에게 요청 
my-first-plugin 과 marketplace.json 을
https://github.com/spacev-team/team-plugins 저장소에 올려줘.

적절한 커밋 메시지로 add·commit 하고,
origin main 으로 push 해줘.
4) 팀원이 설치 — Claude Code 안에서 
/plugin marketplace add spacev-team/team-plugins
/plugin install my-first-plugin@spacev-team
업데이트 전파 — plugin.json(또는 marketplace 항목)의 version을 올려 push하면, 팀원은 /plugin marketplace update 후 새 버전을 받습니다. (version을 생략하면 git 커밋 SHA 기준으로 매 커밋이 새 버전으로 취급됩니다.)
★ Best Practice

이 모든 걸 한 번에 — Claude에게 배포까지 맡기기

구조가 익숙해졌다면, 만들기부터 검증·배포 준비까지 한 번에 요청할 수 있습니다.

방금 만든 my-first-plugin 을 팀에 배포할 수 있게 정리해줘.

1) `claude plugin validate` 로 검증
2) 설치·사용법을 담은 README.md 작성
3) plugin.json 의 version 을 확인해서 정리
4) spacev-team/team-plugins 저장소용 .claude-plugin/marketplace.json 생성

마지막에 팀원이 설치할 때 쓸 명령어도 알려줘.
  • 실제 push는 사람이 확인 후 직접 합니다 — Claude에게는 준비까지 맡기는 게 안전합니다.
  • 업데이트할 때도 “version 올리고 marketplace.json 갱신해줘”처럼 시키면 됩니다.
더 알아보기 — 플러그인에는 훅(hooks/), LSP 서버(.lsp.json), 백그라운드 모니터(monitors/)도 담을 수 있습니다. Claude Code 공식 플러그인 문서에서 전체 구성요소를 확인하세요.
